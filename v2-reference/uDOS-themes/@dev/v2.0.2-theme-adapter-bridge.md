# v2.0.2 Theme Adapter Bridge

This note extends the `v2.0.1` token bridge into starter adapter ownership for
publishing, prose, GUI preview, and gameplay skin surfaces.

## Purpose

Keep visual ownership in one public repo while allowing multiple runtime
consumers to share the same render pipeline.

## Owned Here

`uDOS-themes` now acts as the public source for:

- prose preset maps
- shell mappings
- publishing theme adapters
- email-safe mappings
- gameplay skin adapters that decorate shared render targets

## Current Starter Files

- `src/theme-tokens.json`
- `src/shell-theme-map.json`
- `src/publishing-theme-map.json`
- `src/prose-preset-map.json`
- `src/gameplay-skin-map.json`

## Usage Rule

- runtime repos reference theme ids and prose preset ids
- runtime repos do not become owners of theme packs
- viewer controls such as global font scaling remain surface-local behavior
- compile metadata may declare prose preset and theme adapter refs

## Gameplay Rule

`SKIN` should resolve through theme adapters, not by creating a second render
pipeline. Lens alignment remains advisory unless a later contract explicitly
adds enforcement.
