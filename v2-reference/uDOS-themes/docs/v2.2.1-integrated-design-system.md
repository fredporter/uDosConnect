# uDOS-themes Integrated Design System

This document formalizes `uDOS-themes` as a true cross-surface design system
for the family.

## Core Position

Themes define appearance.
Adapters translate appearance.
Runtimes render appearance.

## Required Surface Coverage

- browser GUI
- ThinUI
- TUI
- workflow/binder surfaces
- publishing and email-safe output
- GTX-form step flows

## Required Theme Layers

1. base tokens
2. component primitives
3. surface adapters
4. themes
5. skins

## Required ThinUI Packs

- `thinui-c64`
- `thinui-nes-sonic`
- `thinui-minimal-safe`

## Required Adapter Lanes

- `browser`
- `thinui`
- `tui`
- `workflow`
- `publish`
- `forms`

## Required Validation

- token completeness
- adapter compatibility
- ThinUI fallback presence
- TUI rendering contract presence
- form flow example presence
- publish HTML and email-safe theme presence

## Upstream fork and demo index

Four integration forks (**fredporter/c64css3**, **fredporter/NES.css**,
**fredporter/svelte-notion-kit**, **fredporter/bedstead**), default **Tailwind
Typography** + **Svelte**, and **GTX forms** boundary: see `theme-upstream-index.md`
and `wiki/credits-and-inspiration.md`.
