# v2.0.2 Runtime Services Kickoff

`v2.0.2` starts by extending the `v2.0.1` foundation spine with explicit
runtime-service contracts.

## New Surface

- runtime services manifest: `#runtime services`
- machine-readable contract artifact: `contracts/runtime-services.json`

## Purpose

The foundation spine answered who owns the platform contracts.

The runtime-services manifest starts answering which shared services sibling
repos can consume during the first working-system round.

The checked-in JSON artifact is the current import path for sibling repos that
need a stable machine-readable source without executing Core directly.

## Starter Service Keys

- `runtime.command-registry`
- `runtime.capability-registry`
- `runtime.release-lanes`

## Notes

- this is still a starter contract, not a full service implementation
- `v2.0.2` builds on `v2.0.1` rather than replacing it
- Shell, Wizard, and product repos should consume these service definitions
  instead of inventing parallel runtime-service catalogs
