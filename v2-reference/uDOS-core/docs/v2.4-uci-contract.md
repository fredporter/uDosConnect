# v2.4 Universal Controller Input Contract

## Purpose

Universal Controller Input (UCI) defines a shared controller event and
prediction contract for Core consumers.

Core owns the canonical event shape, mode model, mapping schema, prediction
contract, and reserved semantic actions. Shell and other product surfaces apply
those contracts in UI-specific ways.

## Shared Mode Set

- `nav`
- `text`
- `command`
- `edit`
- `game`
- `media`

## Event Model

Every controller input should normalize into one event stream with:

- device metadata
- control identifier
- press, release, or repeat phase
- analog values when present
- timestamp
- active mode
- modifier state

## Prediction Model

Prediction should layer low-latency sources first:

1. local static dictionary
2. recent inputs
3. command registry
4. binder vocabulary cache
5. app or domain vocabulary
6. optional local reranker
7. optional remote reranker through Wizard

Remote ranking must never block typing or command acceptance.

## Reserved Semantic Actions

- confirm
- cancel
- back
- menu
- palette
- next tab
- previous tab
- next panel
- previous panel
- accept prediction
- delete backward
- submit

## v2.4 Scope

The first v2.4 slice is shell-first and contract-led. Broader rollout into
`uHOME` and Gameplay stays staged after the initial Core-plus-Shell prototype.
