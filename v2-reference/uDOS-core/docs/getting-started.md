# uDOS-core Getting Started

1. Read `README.md` and `docs/README.md`.
2. Read `docs/boundary.md` and `docs/architecture.md`.
3. Review `docs/activation.md`.
4. Inspect `contracts/` and `schemas/` before changing runtime behavior.
5. Add tests in `tests/` for any contract or semantic change.
6. Run `scripts/run-core-checks.sh` before promotion.
7. Run `scripts/run-contract-enforcement.sh` when changing family boundary or dependency rules.
8. Keep machine-specific environments outside the repo tree.
