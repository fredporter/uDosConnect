# v2.0.3 Render Foundation

`v2.0.3` establishes the starter shared render contract for the public repo
family.

## Purpose

Make Markdown-first publishing, preview, Beacon library output, and GUI render
surfaces build on one compile-first contract instead of repo-local pipelines.

## Ownership

`uDOS-core` owns:

- render document contract shape
- compile artifact metadata used by renderers
- target ids for shared reader-facing outputs
- canonical `md + frontmatter -> semantic html` base transform contract

`uDOS-core` does not own:

- theme packs
- GUI shells
- network publishing orchestration
- provider-backed publish steps

## Starter Contract

Canonical artifact:

- compile creates the canonical binder or content artifact

Shared render step:

- render adapters consume the compiled artifact and output target-specific
  HTML or preview payloads

Starter target ids:

- `web-prose`
- `email-html`
- `gui-preview`
- `beacon-library`

Starter metadata fields:

- `prose_preset`
- `theme_adapter`
- `render_targets`
- `skin_id`
- `lens_id`

## Repo Split

- `uDOS-core` = render contract and semantic transform ownership
- `uDOS-themes` = prose presets, theme adapters, CSS packs, email-safe maps,
  gameplay skin adapters
- `uDOS-wizard` = preview APIs, publish orchestration, Beacon networking-side
  handoff
- `uHOME-server` = nearby reading/library surfaces behind Beacon
- `uDOS-docs` = family explanation and learning material
- private apps = consumers of the shared render pipeline rather than unique
  owners of the contract

## Educational Hook

This round is intended to support one teaching story across:

- Markdown-first compile artifacts
- prose-style web output
- email-safe sibling outputs
- Wizard GUI and Thin GUI previews
- gameplay skin overlays on the same shared theme pipeline
