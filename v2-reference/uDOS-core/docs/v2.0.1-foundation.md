# v2.0.1 Foundation Spine

`v2.0.1` is the first runnable platform-spine round for the current v2 repo
family baseline.

## Purpose

Turn the existing scaffold and boundary docs into explicit starter contracts
that sibling repos can reference without redefining ownership.

## Starter Surfaces

- foundation manifest: `#system foundation`
- release lanes: `#release lanes`
- capability resolution: `#capability resolve <key>`
- starter schemas:
  - `schemas/plugin-manifest.schema.json`
  - `schemas/binder-workflow.schema.json`

## Initial Capability Keys

- `core.runtime.command`
- `shell.adapter.operator`
- `wizard.orchestration.provider`
- `vault.memory.record`

## Notes

- these contracts are intentionally small and starter-grade
- Shell and Wizard may mirror these surfaces but should not redefine them
- `v2.0.2` expands these contracts into working shared runtime services
- the first `v2.0.2` extension starts at `docs/v2.0.2-runtime-services.md`
