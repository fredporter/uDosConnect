# v2.0.4 OK Agent Core Contracts

## Purpose

Define the Core-owned contract layer for v2 OK Agent participation without
transferring runtime authority away from deterministic command and workflow
execution.

This contract slice intentionally separates responsibilities:

- `uDOS-core` owns schema and contract shape
- `uDOS-wizard` owns managed network, provider, MCP routing, and live budgeting
- `uDOS-dev` owns contributor fixtures, policy tests, and promotion harnesses

## New Core Contracts

### `ok-agent-capability-contract`

Artifact:

- `contracts/ok-agent-capability-contract.json`

Schema:

- `schemas/ok-agent-capability-contract.schema.json`

Purpose:

- classify an OK Agent capability participant
- bind autonomy class to explicit budget and audit metadata
- keep model/provider participation subordinate to runtime authority

### `mcp-tool-contract`

Artifact:

- `contracts/mcp-tool-contract.json`

Schema:

- `schemas/mcp-tool-contract.schema.json`

Purpose:

- define typed MCP tool metadata for managed routing
- describe budget, approval, and schedule class surfaces
- preserve typed request/response boundaries across Core and Wizard

### `deferred-packet-contract`

Artifact:

- `contracts/deferred-packet-contract.json`

Schema:

- `schemas/deferred-packet-contract.schema.json`

Purpose:

- represent policy-blocked work as auditable deferred packets
- hand managed work to Wizard queues without hidden execution
- keep approval and scheduling state explicit

### `budget-policy-contract`

Artifact:

- `contracts/budget-policy-contract.json`

Schema:

- `schemas/budget-policy-contract.schema.json`

Purpose:

- define the policy shape consumed by managed budget routing
- keep budget group naming stable across repos
- separate live budget ownership (Wizard) from contract ownership (Core)

## Contract Rule

These contracts define shape and interoperability only.

They do not grant execution authority to models, providers, or MCP transports.
