# v2.4 MDC Conversion Engine

## Purpose

MDC is the canonical intake and normalization lane for file import into
markdown-first vault workflows.

Core owns the deterministic contract for intake, classification, normalized
outputs, and vault routing. Launch behavior, editor opening, and provider-backed
enrichment remain outside Core ownership.

## Locked Rules

- markdown is always generated
- original files are never modified
- outputs must be deterministic for the same input and settings
- large datasets route to SQLite
- small datasets route to JSON

Spreadsheet thresholds:

- `<= 5000` rows: JSON
- `> 5000` rows: SQLite

Document rules:

- PDF is text extraction only in the MVP lane
- PPT direction is Marp-style markdown where supported
- rich media extraction remains deferred beyond the MVP lane

## Supported Types

Native:

- `.md`
- `.mdc`

Data:

- `.csv`
- `.tsv`
- `.json`
- `.xml`
- `.yaml`

Spreadsheet:

- `.xls`
- `.xlsx`

Documents:

- `.docx`

- `.rtf`

Web:

- `.html`

Code and logs:

- `.py`
- `.js`
- `.go`
- `.log`

## Output Contract

MDC normalizes into these artifact classes:

- markdown document output: `.md`
- small structured derivative output: `.json`
- large structured derivative output: `.sqlite`

The original input remains untouched. Only normalized derivatives and their
metadata are eligible for vault persistence.

## Vault Routing

- converted markdown belongs in markdown-first vault lanes
- structured derivatives belong beside the conversion record, not in place of
  the source file
- routing metadata must preserve the source path, classifier result, and output
  artifact list

## v2.4 Scope

The v2.4 lane is contract-plus-MVP conversion behavior. It does not promise a
full rich-media ingestion stack, OCR, image extraction, or live editor launch.
