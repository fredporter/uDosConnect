# uDOS-gameplay Boundary

## Owns

- gameplay-focused modules
- simulation examples
- public teaching artifacts for gameplay patterns
- visual and interactive interpretation of Grid truth
- gameplay world-prototype framing and export rules

## Does Not Own

- runtime semantics
- canonical spatial identity, seed truth, or place persistence
- Core-level spatial vocabulary ownership
- home-server scheduling
- provider bridges
- remote-service supervision
- canonical cloud truth
