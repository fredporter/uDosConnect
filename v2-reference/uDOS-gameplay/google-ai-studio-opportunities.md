# Google AI Studio Opportunities for uDOS-gameplay

## Best opportunities
- multiplayer prototypes
- shared world editors
- realtime crypt or binder placement
- browser-first public world nodes
- collaborative object construction

## Named first prototype

- `multiplayer crypt-placement world`

This is the first gameplay-facing target for the Google MVP round.

The prototype is intentionally small:

- one shared crypt room or shard
- player placement within a bounded map
- cooperative interaction with crypt objects
- extractable world state that can be exported back into gameplay-owned
  artifacts

## Keep grounded
- use AI Studio to accelerate experiments
- extract stable mechanics into reusable uDOS-native contracts
- keep world state portable
- keep canonical gameplay ownership in repo-owned docs, examples, and tests

## Export rules

Generated output must be exportable into gameplay-owned artifacts such as:

- layout examples
- object state examples
- mapping notes from generated fields into gameplay semantics

Gameplay does not own:

- provider routing
- remote-service supervision
- canonical cloud truth
