# `uDOS-empire` Dev Workspace

Use `@dev` only for compact repo planning and current Empire notes.

Keep these lanes small:

- `@dev/requests/` for a short active index
- `@dev/submissions/` for a compact completed summary
- `@dev/notes/` for current repo-owned technical notes
- `@dev/rounds/` only while a live round is still open

Family-wide planning, cross-repo routing, and release coordination stay in
`uDOS-dev/@dev`.
