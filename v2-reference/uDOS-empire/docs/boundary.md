# uDOS-empire Boundary

## Owns

- WordPress plugin runtime and admin UX
- local CRM or contact records represented through WordPress users and user meta
- drag-and-drop browser import flows
- contact dedupe, enhancement, provenance, and append-only notes
- binder and workspace linking on contact records
- WordPress-centred email audience and sending workflows
- intake from other family components into local WordPress contact records
- privacy and consent-aware contact handling on top of WordPress controls

## Does Not Own

- canonical runtime semantics or source-of-truth content contracts
- always-on Git or GitHub execution ownership
- the central family repo store
- host runtime services, scheduling, secrets, or policy enforcement
- the main family AI assistance layer
- generic provider transport infrastructure
- canonical runtime hosting owned by `uDOS-host`
