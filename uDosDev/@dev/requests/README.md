# `@dev/requests`

Use this lane only for a small active index of family-level requests that still
need routing.

Repo-owned binders should move into the owning repo's local `@dev/requests/`
once ownership is clear.

Do not keep large historical request piles here. Distill active cross-repo work
into one compact index and delete stale request files.

## Required Contents

The active index should state:

- title
- requested outcome
- owning repo or stream
- boundary questions
- acceptance criteria
- whether it is single-repo or cross-repo

See `active-index.md`.
