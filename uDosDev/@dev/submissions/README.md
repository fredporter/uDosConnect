# `@dev/submissions`

Use this lane only for a compact summary of meaningful family-level promotion
results.

Repo-owned submissions should live in the owning repo's local
`@dev/submissions/`.

Do not keep large historical submission piles here. Distill the result into one
compact summary and delete stale round files.

## Required Contents

The summary should state:

- binder ID
- owning repo
- summary of promotable output
- validation status
- target branch or promotion path
- open risks or blockers

See `completed-summary.md`.
