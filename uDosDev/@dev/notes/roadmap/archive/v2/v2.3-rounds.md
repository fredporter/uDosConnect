# v2.3 Rounds

Status: completed
Updated: 2026-03-21

## Purpose

The governing spec for this version is:

- `@dev/notes/roadmap/v2.3-unified-spec.md`

v2.3 is the archive-migration and productization lane after the first working
`v2.2` runtime, MCP, and ThinUI promotion. The focus shifts from first
implementation to replacing archive-era dependencies with active v2 family
surfaces so the old archives can be retired.

v2.3 is complete only when:
- Wizard GUI recovery is complete and OK assistant handling is stable
- Ubuntu has a browser-workstation scaffold aligned to the browser-first local
  workstation role
- `@dev` schedules, binders, and automation control are stronger and more explicit
- Sonic live/install/recovery flows are productized enough to replace archive-era dependency
- the archive-removal gate has enough evidence to retire or freeze the old archive repos safely
- the full release matrix validates cleanly

## Locked System Requirements

The following requirements extend every `v2.3` round:

- strict three-lane split: Core, Wizard, and `@dev`
- locked canonical language: `@workspace`, `#binder`, and `@dev`
- binder-centric lifecycle upgraded to `compile -> package -> render -> deploy -> operate`
- Wizard as canonical control plane for publishing, OK agents, API budgeting, and external integration
- dual UI model: ThinUI plus browser workflow UI
- token-based theme system with multi-target output adapters
- formalized `@dev` operating lane with contributor, education, and Goblin test scaffolding expectations
- module-system enforcement in place of legacy extension language
- vault remains source of truth even when services project or cache
- education, publishing, and release surfaces are version requirements, not optional extras
