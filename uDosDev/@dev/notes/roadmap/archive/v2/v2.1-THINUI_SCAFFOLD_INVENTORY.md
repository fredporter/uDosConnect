## v2.1 Round B Scope: Runtime Wiring

### Breakdown Items (from v2.1-rounds.md Round B)

1. **Keep state and event contracts in ThinUI runtime lane**
   - State: already in `src/contracts/state.ts`
   - Events: define `ThinUiEvent` type (action, select, navigate, dismiss, launch-browser, request-refresh)
   - Ownership: ThinUI runtime lane owns these; consumers depend on them

2. **Complete the runtime loop: state hydrate → view resolve → frame render → event emit**
   - Phase 1 (Scaffold): Function signatures and placeholder implementations
   - Phase 2 (Integration): Wire to Core state subscription and Themes adapter
   - Phase 3 (Boot path): Executable boot flow with real rendering

3. **Add `renderThinUiState()` bridge with theme consumption by `state.themeId`**
   - Create `src/index.ts` export function `renderThinUiState(statePacket, themeId)`
   - Function signature:
   ```typescript
   export async function renderThinUiState(
     statePacket: ThinUiStatePacket,
     themeResolver: ThemeResolver
   ): Promise<ThinUiFrame>
   ```
   - Responsibility: Call theme adapter, pass state, receive frame

4. **Implement `resolveThinUiTheme()` in `uDOS-themes/src/adapters/thinui/`**
   - Location: `uDOS-themes/src/adapters/thinui/index.ts`
   - Responsibility: Load adapter by name
   - Signature:
   ```typescript
   export function resolveThinUiTheme(themeId: string): ThinUiThemeAdapter
   ```
   - Return interface:
   ```typescript
   interface ThinUiThemeAdapter {
     renderState(statePacket: ThinUiStatePacket): Promise<ThinUiFrame>;
     getTokens(): ThemeTokens;
     getName(): string;
   }
   ```

5. **Render first adapter-driven boot screen (`thinui-c64`) and document launch flow**
   - Adapter: `uDOS-themes/src/adapters/thinui/c64/index.ts`
   - Visual: Commodore 64 style (blue background, white text, retro fonts)
   - Ability: Render title + subtitle, actions, panel items
   - Launch doc: Create `docs/thinui-boot-launch-sequence.md`

6. **Stage Alpine Thin GUI and sonic-screwdriver Thin GUI launcher contracts for shared consumption**
   - Alpine contract stub: `contracts/alpine-thin-gui.md`
   - Sonic contract stub: `contracts/sonic-thin-gui.md`
   - Both define how ThinUI is invoked from distro launcher
