# v2.0.3 Defaults Vs Examples Rule

Status: active  
Updated: 2026-03-15

## Purpose

Give the family one short placement rule for checked-in starter material during
`v2.0.3`.

## Rule

Use `examples/` when the file is:

- a starter payload
- an operator walkthrough sample
- a sample request or response
- a sample runtime profile intended to be copied, inspected, or tested

Use `defaults/` only when the file is:

- a reusable checked-in default profile
- a compatibility default shared by more than one workflow or component
- intended to be treated as a baseline rather than a one-off example

Do not use `defaults/` just because a file is “useful” or “starter” material.
If it is mainly illustrative, it belongs in `examples/`.

## Related Placement Rule

- `config/` is for checked-in non-secret templates and config examples
- `vault/` is for tracked Markdown-first sample or teaching records
- `memory/` is for mutable runtime state only
